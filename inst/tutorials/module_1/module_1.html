Introduction to R and RStudio

Welcome to Introduction to R and RStudio!

Introduction to the R language

R is a powerful programming language that was designed with statistics and data analysis in mind. It has a rich ecosystem of packages which can be utilized to perform complex data manipulations, analyses and visualizations. Moreover, it is highly extensible and allows you to integrate with other languages (like C++, Python, Julia, etc.) if needed.

Introduction to RStudio

RStudio is an integrated development environment (IDE) for R, providing a convenient and user-friendly interface for writing, running, and debugging R code. It includes several useful features such as syntax highlighting, code completion, and version control integration. RStudio makes working with R easier and more enjoyable. RStudio is often used by I/O psychology practitioners because it simplifies many of the technical aspects of coding, allowing practitioners to focus more on the insights gleaned from the data.

It is quite possible you may be thinking, “so, what’s the difference between R and RStudio? Aren’t they the same thing?” Well, imagine R as a versatile and powerful kitchen appliance like a food processor, while RStudio is like a cookbook filled with recipes and step-by-step instructions on how to use the appliance.

R is the underlying programming language and software environment for statistical computing and graphics. It’s like a food processor that can help you perform various tasks such as slicing, chopping, and blending ingredients. R is what allows you to perform data analysis, create visualizations, and develop statistical models.

On the other hand, RStudio is an integrated development environment (IDE) designed to work with R. It’s like a cookbook that teaches you how to use the food processor (R) effectively, guiding you through the process of preparing a variety of dishes. RStudio provides features such as code editing, debugging, and project management, making it easier to write, run, and organize your R code.

Navigating RStudio

Pane Layout

RStudio’s interface is divided into four main panes:

  • Script Editor: This is where you will write and edit your R scripts. It features syntax highlighting and code completion.

  • Console: This is where R code is executed. You can also type code directly into the console, but it’s often better to write code in the script editor and then run it, so you have a record of your code.

  • Environment/History: The Environment tab shows your current workspace, including all variables, data frames, and functions that exist in memory. The History tab shows your command history.

  • Files/Plots/Packages/Help: This pane allows you to navigate your file system, see plots generated by your code, manage packages, and access R’s built-in help system.

Running Code in the Console

To run code from your script editor, you can simply highlight the code you want to run and press the Run button, or use the shortcut Ctrl+Enter (Cmd+Enter on Mac). The code will be sent to and executed in the console.

Let’s run a simple line of code:

print("Hello, I/O Psychology!")
## [1] "Hello, I/O Psychology!"

After running this code, the console will display the message “Hello, I/O Psychology!”.

Environment Pane

The Environment pane in RStudio is an extremely helpful tool as it displays all the objects (like vectors, data frames, and functions) that you’ve created in your R session. You can see the object’s name, class, length, and content, giving you a quick overview of what’s currently in your workspace.

Let’s create a simple vector and a data frame to see how they appear in the Environment pane:

# create a vector of employee ages
employee_ages <- c(25, 30, 35, 40)

# create a data frame of employees
employees <- data.frame(
  id = 1:4,
  age = employee_ages,
  department = c("HR", "Marketing", "Sales", "IT")
)

Getting Help

As you start your journey with R, you will undoubtedly encounter challenges or unfamiliar tasks. That’s perfectly normal! One of the most important skills in programming, and especially in data analysis, is knowing how to find help when you need it. Let’s explore some of the ways you can get help.

Built-in R Help System

R has a built-in help system that you can access directly from RStudio. If you want to know more about a function, you can use the help() function or the ? shortcut, followed by the name of the function. For example, if we wanted to learn more about the print() function, we could use:

help(print)
# or 
?print

This will open a help page in the Files/Plots/Packages/Help pane, providing detailed information about the function.

Online Resources

While the built-in help system is great for learning about specific functions, you will often encounter more complex problems or need broader advice. That’s where online resources come in handy. Here are a few key ones:

  • Google: Don’t underestimate the power of a well-phrased Google search! Often, someone else has already encountered the same problem and found a solution. Try to be as specific as possible in your search, and include “R” to get more relevant results.

  • Stack Overflow: This is a question-and-answer website for a wide range of programming topics, including R. You can search for your problem, and if it hasn’t been asked before, you can post a new question. Remember to be specific and include a minimal reproducible example if possible.

  • GitHub: Many R packages are developed openly on GitHub, and you can often find help and examples in the repository’s issues or wiki. If you think you’ve found a bug in a package, you can report it in the issues section.

  • Package Documentation and Vignettes: Most R packages come with detailed documentation and vignettes (long-form guides). You can access the documentation by using the help() function or ?, and vignettes can be accessed with the vignette() function.

  • R-Bloggers: This is an aggregate of blogs about R, where you can find tutorials, examples, and advice on a wide range of topics. R-bloggers website.

  • Twitter: Many R users and developers are active on Twitter, using the hashtag #rstats. It’s a great place to ask questions, share your work, and keep up with the latest developments in the R community.

  • Reddit: The Reddit community can also be a helpful resource, particularly the r/rstats and r/RStudio subreddits. You can post questions, share your work, and engage in discussions with other R users. As with other online communities, try to be specific in your questions and provide a reproducible example if possible. Remember, the goal is to make it as easy as possible for others to understand your problem and provide help.

Conclusion

Learning to effectively search for help and solutions online is a valuable skill that will greatly aid your journey into R and data analysis. The more you practice, the better you’ll become at finding the information you need.

Remember, everyone starts from scratch and it’s okay not to know everything. The R community is known for being welcoming and helpful, so don’t hesitate to ask questions when you’re stuck!

Project Workflow and Reports using Quarto

Organize Your Work Using RStudio Projects

Working with data often involves multiple files: data files, code files, and output files. Keeping these files organized can be a challenge, especially as a project grows in complexity. RStudio Projects are a powerful tool to keep your work organized.

[Image Placeholder or instructions to create new project]

When you create a new RStudio Project, RStudio creates a new directory/folder on your computer. This new directory will contain an .Rproj file which stores settings related to your project. When you open this .Rproj file, RStudio will start a new session with the working directory set to the directory containing the .Rproj file. This means that you can use relative paths (paths that do not start with / or ~) to your files which makes your project more portable.

For example, instead of using a full path to read a data file:

data <- read_csv("/Users/yourname/Projects/myproject/data/mydata.csv")

You can use a relative path, i.e., the path starting from your project myproject directory:

data <- read_csv("data/mydata.csv")

This way, as long as the relative structure of your files remains the same, you can move your project directory or even share it with others, and your code will still work!

Creating a New Quarto Document

Quarto offers a comprehensive framework for data science authoring, integrating your code, its results, and your written content. Quarto documents are fully reproducible and support various output formats, including HTML web pages, PDFs, Word files, presentations, and more.

Quarto documents serve three primary purposes:

  • Communication with decision-makers: These individuals prioritize understanding the key findings rather than delving into the technical aspects of the analysis.

  • Collaboration with fellow data analysts/scientists: These individuals are interested in both the conclusions drawn and the underlying code that led to those conclusions.

  • Data science environment: Quarto serves as a contemporary lab notebook, allowing you to document not only your actions but also your thoughts during the data science process.

To use Quarto with R, you should install the rmarkdown R package:

install.packages("rmarkdown")

Installation of the rmarkdown package will also install the knitr (we’ll come back to knitr later) package so you will have everything required to render documents containing R code.

Markdown Basics

Markdown is a simple language for formatting text. You can create headings by starting a line with one or more # symbols (e.g., # Heading 1, ## Heading 2). You can create lists by starting lines with - or *. You can make text bold or italic by surrounding it with ** or * (e.g., **bold text**, *italic text*).

You can include links using the following syntax:

[link text](https://www.example.com)

And you can include images using a similar syntax:

![image alt text](path/to/image.jpg)

Using the Visual Editor with Quarto

But what if you are new to markdown and prefer a simplier way to format your document? Well you are in luck!

In addition to the classic source editor (which is what you see when you first create your new Quarto document), RStudio provides a visual editor that provides a live preview of your document as you write it. The visual editor can be a great aid when you’re getting started with Quarto and markdown, as it allows you to see the results of your formatting immediately, rather than having to knit the document to see the final output.


[Image placeholder]

The visual editor provides several features that can make your workflow more efficient:

  • Live Preview: As you type, you can see a live rendering of your document. This includes text formatting, code chunks, images, tables, and other elements of your document.

  • WYSIWYG Editing: You can apply text formatting (like bold or italics), create lists, insert links, and more using the toolbar at the top of the editor. You can also insert code chunks and other special elements using the Insert button on the toolbar.

  • Code Chunk Controls: Code chunks in the visual editor come with a set of controls that allow you to run the code, display the output inline, or hide the code or output.

  • Outline View: The visual editor provides an outline view of your document, which allows you to quickly navigate to different sections.

To switch to the visual editor in RStudio, you can use the dropdown menu in the top-right corner of the source editor pane. Select “Visual Editor” from this menu, and RStudio will switch to the visual editor for the current document.

Remember, the visual editor is just another way to interact with your Quarto document. Whether you use the visual editor or the source editor is a matter of personal preference and depends on your workflow. You can switch between the two at any time without affecting your document.

Code Chunks

In Quarto, code chunks are the parts of the document where you can include code from various programming languages, such as R, Python, and SQL. These chunks can be used to perform computations, create visualizations, and more. The output of the code chunk (if any) is then included directly in the document.

Here’s a basic example of a code chunk in Quarto:

print("Hello world!")

When you knit your document, the code in the code chunk will be executed and the results will be included in the output document.

You can control the behavior of code chunks with chunk options, which are specified in the {r} part of the chunk header.

Here are some of the most commonly used chunk options:

  • echo: Determines whether the code in the chunk should be included in the final document. If echo=FALSE, the code will be executed, but the code itself won’t be shown in the document, only its output.

  • eval: Determines whether the code in the chunk should be executed. If eval=FALSE, the code won’t be executed, and no output will be included in the document.

  • include: Determines whether the chunk (both the code and the output) should be included in the final document. If include=FALSE, the chunk will be completely ignored when the document is rendered.

  • fig.cap: Provides a caption for any figures that are produced by the code chunk.

Quarto introduces a new way to specify chunk options (well, new if you have not used R Markdown before). Instead of including them on the same line as the backticks and language identifier, you can include them in special comments at the top of the code chunk. For example:

In this example, echo: false is a chunk option that tells Quarto not to include the code in the final output, and fig-cap: "Air Quality" is a chunk option that provides a caption for a figure produced by the code chunk.

This way of specifying chunk options makes it easier to include longer options like fig-cap, fig-subcap, and fig-alt. It also improves compatibility with more structured editors that don’t have an easy way to edit chunk metadata (e.g., most traditional notebook UIs).

That being said, if you prefer, you can still include chunk options on the first line (e.g., {r, echo = FALSE}). However, we recommend using the comment-based syntax to make your documents more portable and consistent across execution environments.

For more detailed information about Quarto chunk options, you can refer to the chunk options documentation.

YAML and Output Formats

At the top of a Quarto document, you’ll find a block of text surrounded by ---. This is the YAML metadata for the document. It specifies options for the document, such as the output format. For example, here’s a YAML block for a document specifying the title and that the output should be in HTML format:

---
title: "My Document"
format: html_document
---

The format field specifies the output format. You can specify multiple formats by listing them:

---
title: "My Document"
format: 
  - html_document
  - pdf_document
---
